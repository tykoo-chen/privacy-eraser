# 🛡️ ErasePAST — 你的个人公关 Agent

> 像明星经纪人一样管理你的数字身份。扫描全网 → 友好协商 → 平台举报 → SEO 稀释 → 持续监控。

ErasePAST 是一个 **AI Agent Skill**，用户只需要输入姓名，Agent 会自动执行全套隐私保护流程。不是一个网页工具，是一个**能思考、能操作、能跟进**的 Agent。

---

## 核心理念：先礼后兵

```
用户说名字 → Agent 全网搜 → 分类展示 → 用户确认 → Agent 自动执行
```

**不是暴力删除，是像公关团队一样处理：**

- 先友好私信发布者，请求删除
- 对方不回？发正式函，引用法律条文
- 还不行？平台举报
- 举报没用？SEO 稀释，用正面内容压下去
- 实在不行？法律手段

**关键：Skill 之间不是线性流水线，是有反馈回路的。** Negotiator 失败会升级到 Reporter，Reporter 失败会升级到 Legal Action，SEO Diluter 和其他 Skill 可以并行。Monitor 持续运行，发现新问题会重新触发整个流程。

### 7 个专业 Skill

| Skill | 做什么 | 什么时候触发 | 失败后升级到 |
|-------|--------|------------|------------|
| 🔍 **Scanner** | 全网搜索，分类标记风险 | 任务开始 / Monitor 发现新内容 | — |
| 💬 **Negotiator** | 友好私信发布者 | 有明确发布者 | → 🚨 Reporter |
| 🚨 **Reporter** | 平台举报 + 快照删除 | 协商无果 / 匿名内容 | → ⚖️ Legal |
| 📈 **SEO Diluter** | 正面内容压制负面结果 | 删不掉的内容（并行启动） | — |
| 🗑️ **Data Broker** | 数据经纪人 Opt-out | 发现数据被交易 | — |
| ⚖️ **Legal Action** | 律师函 / 监管投诉 | 所有手段无效 | — (终点) |
| 📊 **Monitor** | 每周扫描 + 效果追踪 | 持续运行 | → 重新触发 Scanner |

## 执行流程

### 阶段一：扫描 & 用户决策

```
┌──────────────────────────────────────────────────────────────┐
│                      🔍 Scanner Skill                        │
│                                                              │
│  web_search "姓名"                                           │
│  web_search "姓名" site:zhihu.com / weibo.com / ...          │
│  web_search "姓名" 手机 OR 简历 OR 身份证                      │
│                                                              │
│  输出：                                                       │
│  🔴 手机号泄露 ×3    🟡 负面/过时 ×2    🟢 正面 ×5    ⚪ 无关 ×2  │
└──────────────────────────────┬───────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────┐
│                     🧑 用户确认处理方向                        │
│                                                              │
│  "手机号的都处理"  /  "全部处理"  /  "1和3处理，其他不管"        │
│                                                              │
│  → 用户只做这一个决策，后面全部自动                              │
└──────────────────────────────┬───────────────────────────────┘
                               │
                               ▼
```

### 阶段二：策略路由（核心逻辑）

根据每条内容的**类型 × 来源**，路由到不同 Skill **并行执行**：

```

                    │  🎯 策略路由引擎      │
                    │  (Orchestrator)      │
                    └──┬──┬──┬──┬──┬──┬───┘
                       │  │  │  │  │  │
        ┌──────────────┘  │  │  │  │  └──────────────┐
        │     ┌───────────┘  │  │  └────────┐        │
        │     │     ┌────────┘  └─────┐     │        │
        ▼     ▼     ▼                 ▼     ▼        ▼
     ┌─────┬─────┬─────┬─────────┬──────┬────────┐
     │情况1│情况2│情况3│  情况4   │情况5 │ 情况6  │
     └──┬──┴──┬──┴──┬──┴────┬────┴──┬───┴───┬────┘
        │     │     │       │       │       │
        ▼     ▼     ▼       ▼       ▼       ▼

  隐私泄露  隐私泄露  负面评价  负面评价  过时信息  数据经纪
  有发布者  匿名/无人  不实     属实/大V  旧简历等  人交易
     │        │       │        │        │       │
     ▼        ▼       ▼        ▼        ▼       ▼
   💬协商   🚨举报   💬协商    📈稀释   💬协商   🗑️清理
   (48h)    (直接)   +证据     (并行)   请求更新  Opt-out
     │                (72h)              │
     │ 无果            │ 无果            │ 无果
     ▼                ▼                 ▼
   🚨举报           🚨举报            🚨举报
     │                │                 │

```

**注意：**
- 多条内容**同时**走不同路径，并行执行
- 📈 SEO Diluter 不需要等其他 Skill 失败，**从 Day 0 就可以并行启动**
- 每条路径有独立的升级时间线

## 策略决策矩阵

| 内容类型 | 有发布者 | 匿名内容 | 自己可编辑 |
|---------|---------|---------|-----------|
| 手机号/身份证泄露 | 协商 → 举报 | 直接举报 | 直接删除 |
| 负面评价（不实） | 协商+证据 → 举报 | 举报诽谤 | 直接删除 |
| 负面评价（属实） | 协商 → SEO稀释 | SEO稀释 | — |
| 过时简历/资料 | 协商更新 | 举报 → 快照删除 | 直接更新 |
| 大V发布的内容 | **谨慎！** 优先SEO | SEO稀释 | — |

### 各策略成功率

| 策略 | 成功率 | 时间 |
|------|--------|------|
| 友好协商 | 40-60% | 1-3 天 |
| 平台举报（隐私类） | 70-90% | 1-7 天 |
| 平台举报（负面评价） | 30-50% | 3-15 天 |
| 搜索快照删除 | 80%+ | 3-15 天 |
| SEO 稀释 | 渐进 | 2-8 周 |
| 法律函件 | 80%+ | 3-7 天 |

## 怎么用

### 方式一：作为 AI Agent Skill（推荐）

把 `SKILL.md` 加载到你的 AI Agent（OpenClaw / Claude / 其他支持 Skill 的 Agent），Agent 会按照指令自动执行。

```
你：帮我清理一下网上的个人信息
Agent：好的，告诉我你的名字？
你：张三
Agent：正在搜索... 找到 12 条结果...
      🔴 百度文库上有你的简历，包含手机号
      🟡 微博上有人发了负面评价
      你想怎么处理？
你：手机号相关的都处理，微博的不管
Agent：收到。已私信百度文库上传者请求删除，
      同时已提交平台举报。我会持续跟进。
```

### 方式二：Python CLI

```bash
# 添加案例
python tools/eraser.py add "https://wenku.baidu.com/xxx" --desc "旧简历包含手机号"

# 查看所有案例
python tools/eraser.py list

# 更新状态
python tools/eraser.py update 1 contacted --note "已私信上传者"
```

### 方式三：Web Demo

```bash
npm install
npm run dev
# 打开 http://localhost:3000
```

Web 界面是交互演示，展示完整流程。用于展示和理解系统逻辑。

## 当前状态：什么能做，什么不能做

### ✅ 现在能做的

| 能力 | 说明 |
|------|------|
| **全网搜索** | 通过 web_search 在搜索引擎和各平台搜索用户信息 |
| **结果分类** | 自动判断 🔴🟡🟢⚪ 风险等级 |
| **生成私信** | 根据场景选择友好/正式/法律模板，填入具体信息 |
| **浏览器操作** | 通过 Playwright / Agent 浏览器能力执行私信、举报 |
| **快照删除申请** | 自动提交百度/Google/Bing 快照删除 |
| **案例管理** | CLI 工具跟踪每条内容的处理状态 |
| **策略决策** | 根据内容类型和来源自动选择最优处理策略 |
| **升级逻辑** | 协商无果自动升级到举报，举报无果建议法律 |
| **14 套模板** | 覆盖协商、举报、法律各场景 |
| **交互演示** | Web Demo 展示完整流程 |

### ⚠️ 能做但依赖条件

| 能力 | 依赖什么 |
|------|---------|
| **真实搜索** | 需要 SerpAPI Key 或 Agent 的 web_search 工具 |
| **浏览器自动化** | 需要 Agent 有浏览器操作能力（如 OpenClaw browser MCP） |
| **平台私信** | 需要目标平台的账号登录状态 |
| **平台举报** | 需要在目标平台有账号 |
| **定期监控** | 需要 Agent 支持定时任务（cron） |

### ❌ 目前不能做的

| 限制 | 原因 |
|------|------|
| **自动登录平台** | 安全原因，需要用户提前登录 |
| **处理验证码** | 无法自动过验证码 |
| **保证删除成功** | 取决于平台响应，不可控 |
| **代替律师** | 法律模板仅供参考，不构成法律建议 |
| **SEO 自动发文** | 只能建议用户发布方向，不能替用户发文 |
| **数据经纪人自动清理** | 大部分经纪人需要手动 Opt-out |
| **跨境法律执行** | GDPR/CCPA 请求的实际执行力有限 |

## 法律依据

本项目基于以下法律法规保护用户隐私权益：

- **《个人信息保护法》** 第四十七条：个人有权请求删除个人信息
- **《民法典》** 第一千零三十四条：个人信息受法律保护
- **《网络安全法》** 第四十四条：禁止非法获取个人信息
- **GDPR** Article 17：Right to Erasure（被遗忘权）
- **CCPA** §1798.105：消费者有权要求删除个人信息
